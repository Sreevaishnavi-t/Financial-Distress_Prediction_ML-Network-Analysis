================================================================================
PROJECT OVERVIEW & PRESENTATION CHEATSHEET
A Hybrid Network Analysis and Machine Learning Model for Enhanced Financial Distress Prediction
================================================================================

This document outlines the core technologies, libraries, and machine learning models used in this project to help you prepare for your college presentation.

--------------------------------------------------
1. CORE TECHNOLOGIES (THE TECH STACK)
--------------------------------------------------
- Python: The primary programming language used for both backend logic and data science modeling.
- Django (Web Framework): The backend framework used to create the web application. It handles user requests, routing (URLs), and rendering the HTML templates.
- SQLite: The default relational database used by Django to store application data (like user registrations and logins).

--------------------------------------------------
2. FRONTEND TECHNOLOGIES (USER INTERFACE)
--------------------------------------------------
- HTML5 & CSS3: Used for creating the structure and styling of the web pages.
- JavaScript & jQuery: Used for client-side interactions and dynamic UI elements (like mobile menus and "back to top" buttons).
- FontAwesome: An icon library used to provide vector icons and social logos across the web application.

--------------------------------------------------
3. MACHINE LEARNING & DATA SCIENCE LIBRARIES
--------------------------------------------------
- Pandas: An essential data manipulation library. Used extensively in `views.py` to load the dataset (`fd.csv`), fill missing values (`fillna`), and extract feature columns (`A13` to `A24`) and the target label.
- Scikit-Learn (sklearn): The core machine learning library used in the project. It provides all the classification algorithms, data splitting utilities (`train_test_split`), and evaluation metrics (`accuracy_score`, `confusion_matrix`).
- Imbalanced-Learn: A library used to handle datasets where the classes are not equally distributed (which is common in financial distress prediction, as there are usually far fewer "distressed" companies than healthy ones).
- Matplotlib & Seaborn: Python graphing libraries used for data visualization and plotting graphs.
- Numpy: Used under the hood by pandas and scikit-learn for high-performance mathematical and array operations.

--------------------------------------------------
4. MACHINE LEARNING MODELS IMPLEMENTED
--------------------------------------------------
The project implements several different machine learning classifiers so that their performance/accuracy can be compared. If your professor asks which algorithms are used, mention these:

1. Random Forest Classifier: An ensemble learning method that creates multiple decision trees and merges them together to get a more accurate and stable prediction. It's the primary model used in the application to make actual predictions based on user inputs.
2. K-Nearest Neighbors (KNN): Used (in the SVM view) to demonstrate another classification approach. It classifies new data points based on their distance to neighboring data points in the training set.
3. Decision Tree Classifier: A model that makes decisions based on a tree-like graph of conditions. 
4. Passive-Aggressive Classifier: An online learning algorithm that remains passive for correct classifications and becomes aggressive when making a miscalculation, updating itself.
5. Logistic Regression: A fundamental statistical model used for binary classification (predicting whether a company is in distress or not).

--------------------------------------------------
5. APPLICATION WORKFLOW (HOW IT WORKS)
--------------------------------------------------
If asked to explain the data pipeline, here is the basic flow:
1. Data Loading: The application reads historical financial data from `fd.csv` using Pandas.
2. Pre-processing: Missing data is filled with zeros.
3. Feature Extraction: The features (input variables, typically financial ratios represented as `A13`-`A24`) and target 'LABEL' (distress or no distress) are separated.
4. Train/Test Split: The dataset is split into training data (80%) and testing data (20%).
5. Training: The chosen Machine Learning algorithm is trained on the 80% data.
6. Prediction & Evaluation: The model predicts the outcomes for the remaining 20% test data, and the accuracy score is calculated and displayed on the frontend.
7. User Interaction: The web app also takes live input from the user (12 parameters), processes it through the trained Random Forest model, and instantly predicts if the inputs represent a "financial distress" scenario.
